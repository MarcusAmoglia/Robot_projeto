🚨 ERROS COMUNS E SOLUÇÕES
Erro	Causa	Solução
AttributeError: startswith	Dicionário no lugar do endpoint	Colocar string (URL) no lugar certo
Keyword 'When...' not found	Espaço/TAB extra depois do When	Deixar APENAS 1 espaço
Variable '${x}' not found	Esqueceu de compartilhar	Usar Set Test Variable
Status Should Be com erro	Fez ${var} Status Should Be	Remover ${var} =
📍 ENDPOINTS DO JSONPLACEHOLDER
Endpoint	Método	O que faz
/users/1	GET	Busca usuário ID 1
/posts	POST	Cria novo post (retorna ID 101)
/posts/1	PUT	Substitui post ID 1 inteiro
/posts/1	PATCH	Altera só alguns campos do post ID 1
/posts?userId=1	GET	Busca posts do usuário ID 1
/posts/1	DELETE	Deleta post ID 1
📊 CÓDIGOS DE STATUS
Status	Significado
200	OK (GET, PUT, PATCH, DELETE funcionaram)
201	Criado (POST funcionou)
400	Requisição ruim (você errou algo)
404	Não encontrado
500	Erro no servidor (não é sua culpa)
🧠 CONCEITOS IMPORTANTES
Diferença entre PUT e PATCH
PUT: Substitui o recurso INTEIRO. Precisa enviar todos os campos.

PATCH: Altera APENAS os campos enviados.

Query String (filtro)
Aparece na URL depois do ?

Exemplo: /posts?userId=1 (filtra posts do usuário 1)

Escopo de variáveis
Variável criada dentro de uma keyword MORRE nela

Para usar em outra keyword → precisa de Set Test Variable

Métodos HTTP
Método	Ação
GET	Ler/buscar
POST	Criar
PUT	Substituir tudo
PATCH	Alterar parte
DELETE	Remover
